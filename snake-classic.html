Snake Classic

Snake is one of those instantly recognizable games for everyone. It brings back a nostalgia that is unreplicable. I wanted to remake this game but with my own custom features. The project is a rendition of the classic Snake game developed using the LibGDX framework. The game is designed to be played in a grid where the snake moves around, consuming apples to grow in size, while avoiding collisions with itself, rocks, and rotten apples. (hint those are the new features)

Key Features

1. Game Initialization

The game is launched using the GameLauncher class. Screen dimensions are set using constants for width and height. The game instance is created using the Snake class.

2. Game Components

Snake: The snake is represented by a head (SnakeHead) and a body (SnakePart). The snake moves in different directions based on user input.

Apple: Represented by the Apple class. When the snake consumes an apple, it grows in size.

Rock: Represented by the Rock class. Colliding with a rock ends the game.

Rotten Apples: Represented by the RottenApples class. Consuming a rotten apple clears the snake’s body and resets the score. (You definitely do not want to hit this...)

3. Game Mechanics

The snake’s movement is controlled using arrow keys. The game checks for collisions with apples, rocks, rotten apples, the snake’s body, and the game’s boundaries. The game keeps track of the player’s score and high score, which is saved using Preferences. The game features a grid background to represent the playing field.

4. Graphics and Visualization

The game uses ShapeRenderer for drawing game components. The snake’s head is represented using a texture image (snakeHeadPic). The game has a menu screen with a “START!” button. The game displays the current score and high score on the screen.

5. Game Flow

The game starts with a menu screen. Upon clicking “START!”, the game begins. The game continues until the snake collides with a rock, the game’s boundaries, or itself. If the game ends, the player is returned to the menu screen.

6. Utility Classes

Constants: A class that holds various constants used throughout the game, such as screen dimensions and grid size.

Direction: An enumeration representing the possible directions the snake can move in.
