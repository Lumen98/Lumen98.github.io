FPS_Booster

I wanted to get experience with networking using a client-server-based architecture. I ended up choosing a calculator for its simplicity and flexibility given I can add any function that comes to mind. As such, I created a calculator that starts a server and communicates with the client.

Key Features

1. User Interaction through GUI

The application, built using JavaFX, offers a user-friendly interface with text fields for input and labels for displaying results. Users can perform basic arithmetic operations using dedicated buttons. A history feature displays previous calculations, as well as the maximum and minimum results obtained. A timer showcases the application’s runtime.

2. Calculator Functionalities

The Calculator class handles arithmetic operations such as addition, subtraction, multiplication, and division. A LinkedList maintains a history of calculations, allowing users to revisit past operations. The application also tracks the maximum and minimum answers from the calculations.

3. Visualization and GUI Handling

The Client class manages the GUI creation and event handling. It contains methods to set up the button grid and handle button logic. The TimerThread class updates a label to display the application’s runtime.

4. Networking and Logging

The Network class manages network communication, connecting to a server, sending log requests, and receiving log responses using sockets. The Server class waits for client connections, processes incoming log requests, and sends back log responses. It operates on port 4444 as an example. LogRequest and LogResponse classes facilitate logging, utilizing the Jackson library for JSON serialization and deserialization.

5. Data Handling and Storage

The IOHelper class offers methods to read from and write to an external file, “output.txt”, ensuring a persistent history of calculations.

6. Error Handling and Robustness

The system is designed to handle potential errors and exceptions, ensuring smooth user experience and data integrity.
