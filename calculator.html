Calculator

I wanted to get experience with networking using a client-server-based architecture. I ended up choosing a calculator for its simplicity and flexibility given I can add any function that comes to mind. As such, I created a calculator that starts a server and communicates with the client.

Key Features

1. User Interaction through GUI
The application, built using JavaFX, offers a user-friendly interface with text fields for input, labels for results, and dedicated buttons for arithmetic operations. A history feature displays previous calculations, as well as maximum and minimum results obtained. A timer showcases the application’s runtime.
2. Calculator Functionalities
The Calculator class handles addition, subtraction, multiplication, and division. A LinkedList maintains a history of calculations, enabling users to revisit past operations. The application also tracks the maximum and minimum answers from the calculations.
3. Visualization and GUI Handling
The Client class manages GUI creation and event handling. It sets up the button grid and logic, while the TimerThread class updates a label to display the application’s runtime.
4. Networking and Logging
The Network class manages communication with a server. The Server class waits for client connections, processes incoming log requests, and sends back log responses. LogRequest and LogResponse facilitate logging, leveraging Jackson for JSON serialization and deserialization.
5. Data Handling and Storage
The IOHelper class provides methods to read from and write to an external file (“output.txt”), ensuring a persistent history of calculations.
6. Error Handling and Robustness
The system gracefully handles potential errors and exceptions, ensuring a smooth user experience and data integrity.
