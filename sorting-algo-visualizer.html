Sorting_Algo_Visualizer

This project is a console-based application in Java that aims to visually represent the execution of various sorting algorithms on different sets of integer arrays. The sorting algorithms included are Merge Sort, Bubble Sort, Heap Sort, Selection Sort, and Insertion Sort.

Key Features

1. User Interaction through Console
The user can choose a sorting algorithm to visualize. The user can generate a random array, make a custom array, or use a previously generated array to sort. Upon choosing an algorithm, the system will display the array’s sorting process and state how many iterations it took for the array to be successfully sorted. The application loops until the user decides to exit.

2. Sorting Algorithms
The system includes classes for each of the following sorting algorithms:

- Merge Sort
- Bubble Sort
- Heap Sort
- Selection Sort
- Insertion Sort

3. Visualization
The VisualizerAlgo class is responsible for the visualization of the sorting process. It represents each number in the array as a column of ‘*’ characters, with the height corresponding to the number’s value. The visualization uses a grid of strings, where each number is translated into a column of ‘*’. There’s a counter (iterationCtr) to keep track of the number of iterations, which is displayed after sorting.

4. Array Generation and Handling
The user can generate a random array with values between 0 to 100, and a length ranging from 50 to 100. The user can also input custom array values. Arrays can be copied, displayed, and used for sorting.

5. Error Handling
The system checks for invalid inputs and throws exceptions if needed (e.g. when finding the largest number in an empty or null array).

Conclusion
This Sorting Algorithm Visualizer is a comprehensive tool for users to understand the inner workings of popular sorting algorithms. By providing a visual aid in the form of a console grid, users can see the step-by-step changes in the array as it gets sorted, enhancing the understanding and appreciation of algorithm complexities.
